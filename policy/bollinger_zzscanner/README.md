# 布林带 Z-Score A股扫描器

布林带均值回归策略扫描器，扫描全市场A股，筛选 Z-Score 超跌/超买的股票，结合财务指标（资产负债率）和消息面进行综合评分。

---

## 快速开始

```bash
cd bollinger_zzscanner
python bollinger_zscore_scanner.py
```

依赖：`akshare`、`pandas`、`numpy`

---

## 核心算法

```
Z-Score = (当前收盘价 - N日均线) / N日价格标准差
N = bb_window（布林带窗口，默认20日）
```

---

## 可配置参数

直接修改脚本顶部的 `CONFIG` 字典即可，无需改动其他代码。

### 市场参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `market` | `"A"` | 市场范围：`"A"` 沪深A股 / `"SH"` 上证 / `"SZ"` 深证 |

### 布林带参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `bb_window` | `20` | 布林带窗口（日） |
| `bb_std` | `2.0` | 布林带标准差倍数 |
| `lookback_months` | `60` | 历史回看区间（月），约5年 |

### Z-Score 筛选条件

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `zscore_min` | `-999` | Z-Score 下限，低于此值纳入（`-999` 表示关闭） |
| `zscore_max` | `-2.5` | Z-Score 上限，高于此值纳入（`999` 表示关闭） |

**典型用法：**

```python
# 筛选超跌（价格大幅低于均线）
"zscore_min": -999, "zscore_max": -2.5

# 筛选超买（价格大幅高于均线）
"zscore_min": 2.5, "zscore_max": 999

# 同时筛选超跌和超买
"zscore_min": -2.5, "zscore_max": 2.5
```

### 排序与数量

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `sort_by` | `"buy_score"` | 排序字段：`zscore` / `buy_score` / `return_60d` / `debt_ratio` / `price` |
| `ascending` | `True` | `True` 升序 / `False` 降序 |
| `top_n` | `50` | 最多返回 N 条结果 |

### 基础过滤

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `min_price` | `1.0` | 最低股价过滤（元） |
| `exclude_st` | `True` | 是否排除 ST 股 |
| `score_min` | `0` | 综合评分最低门槛，低于此值不显示 |

### 候选股初筛

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `prefilter_return_days` | `60` | 初筛使用多少日的涨跌幅 |
| `prefilter_return_pct` | `-5` | 初筛阈值（%），N日涨跌幅低于此值才进入精确计算 |
| `max_candidates` | `5000` | 最大候选股数量，防止运行时间过长 |

### 新闻参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `news_count` | `5` | 每只股票获取的新闻条数 |

## 输出文件

| 文件 | 说明 |
|------|------|
| `results.csv` | 原始数据 CSV，字段为英文 key |
| `results.json` | 结构化 JSON，结构如下 |

**JSON 结构：**

```json
{
  "fields": {
    "中文列名1": "key1",
    "中文列名2": "key2",
    ...
  },
  "lists": [
    {"code": "301040", "name": "中环海陆", ...},
    ...
  ]
}
```

- `fields`：字段映射表，`中文名 → 英文key`，用于前端展示列名
- `lists`：数据数组，每条记录用英文 key，与 CSV 表头一致

## 综合评分规则（满分100分）

| 维度 | 满分 | 说明 |
|------|------|------|
| 技术面 | 35分 | Z-Score 越低（超跌越深）分数越高 |
| 基本面 | 35分 | 资产负债率合理 + ROE 为正 |
| 消息面 | 30分 | 近期利好消息越多分数越高，利空扣分 |

**技术面评分（Z-Score 每 0.5 一档，共12档，极度超跌得35分，极度超买得0分）：**

| Z-Score 区间 | 技术分 |
|-------------|--------|
| z ≤ -2.5 | **35** |
| -2.5 < z ≤ -2.0 | 33 |
| -2.0 < z ≤ -1.5 | 31 |
| -1.5 < z ≤ -1.0 | 29 |
| -1.0 < z ≤ -0.5 | 27 |
| -0.5 < z ≤  0.0 | 24 |
|  0.0 < z ≤  0.5 | 20 |
|  0.5 < z ≤  1.0 | 16 |
|  1.0 < z ≤  1.5 | 13 |
|  1.5 < z ≤  2.0 | 10 |
|  2.0 < z ≤  2.5 | 6 |
|  2.5 < z ≤  3.0 | 3 |
|  z > 3.0 | 0 |

**基本面评分（资产负债率）：**

| 资产负债率 | 财务分 |
|-----------|--------|
| < 50% | 35 |
| 50–60% | 30 |
| 60–70% | 25 |
| 70–80% | 18 |
| 80–90% | 10 |
| ≥ 90% | 5 |

ROE > 20% 额外 +5，ROE < -10% 扣 15 分。
